AWS Learning Project - EC2 & Apache

üìã What I Built

This project helped me understand core AWS concepts through hands-on implementation. I launched an EC2 instance (Amazon Linux 2), connected to it using SSH with a key pair, installed Apache web server, and hosted a simple HTML website. It was exciting to see my website accessible from the internet!

üéì What I Learned

  • How to launch an EC2 instance using the AWS Console
  • Choosing instance types (I used t2.micro for Free Tier)
  • Creating and using SSH key pairs for secure access
  • Configuring security groups to allow HTTP (port 80) and SSH (port 22)
  • Installing Apache using sudo yum install httpd
  • Starting the web server with sudo systemctl start httpd
  • Creating simple HTML pages in /var/www/html/
  • Accessing the website using the instance's public IP address

üí° Challenges & Solutions

  • Challenge: Couldn't connect via SSH initially
    Solution: Realized I forgot to add SSH (port 22) to security group. Fixed it and could connect!
  • Challenge: Website wasn't accessible from browser
    Solution: Security group was missing HTTP (port 80). Added it and website started working
  • Challenge: Apache wouldn't start
    Solution: Found I needed to use sudo. Learned about Linux permissions!

üöÄ Next Steps

I want to add HTTPS with SSL certificate, try connecting a database, and maybe use a load balancer. Still learning, but this project gave me confidence!

Security Groups Practice

Security Learning - Firewall Configuration

üìã What I Practiced

Spent time understanding how security groups work as virtual firewalls. Created different security groups for web servers, practiced allowing/denying traffic, and learned about inbound/outbound rules.

üéì What I Learned

  • Security groups act as instance-level firewalls
  • They are stateful - return traffic is automatically allowed
  • Configuring inbound rules to allow SSH (22), HTTP (80), HTTPS (443)
  • Using source IP ranges (0.0.0.0/0 for public, specific IPs for restricted access)
  • Best practice: Only open ports you actually need
  • Testing connectivity with different security group rules

üí° Key Takeaways

Security groups are really important for protecting resources. I learned to be careful about opening ports to 0.0.0.0/0 (internet) and only allow what's necessary. Still learning about best security practices!
